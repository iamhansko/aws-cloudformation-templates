# Cognito Identity Pool

#### 사용자 풀은 연동 및 로컬 사용자 프로필의 디렉터리이며, 사용자에게 인증 옵션을 제공합니다.
- 사용자 풀은 디렉터리, 통합 인증(SSO) 엔드포인트 및 OAuth 2.0 서버입니다. 사용자는 OAuth 2.0 토큰을 생성하여 서버, API 및 사용자에게 직접 안전하게 전달하는 Cognito를 통해 자신을 인증할 수 있습니다. Cognito는 리소스가 토큰 클레임을 권한 부여로 처리하는 데 도움이 되는 여러 SDK와 통합됩니다.
- 사용자 풀 내의 사용자는 사용자 이름과 암호를 통해 Cognito만 사용하거나 OAuth 2.0, SAML 또는 OIDC 자격 증명 공급자의 자격 증명을 함께 사용하여 로그인할 수 있습니다. 또한 Cognito 사용자 풀은 Google, Facebook, Amazon, Apple과 같은 소셜 자격 증명 공급자와의 페더레이션을 지원합니다.
- Cognito를 사용하여 호스팅 UI로 통합 인증(SSO)을 위한 웹 프런트 엔드를 호스트하고 관리할 수 있습니다. 호스팅 UI에는 가입, 로그인, 암호 재설정, 멀티 팩터 인증(MFA) 사용을 위한 사용자 흐름이 포함됩니다. 또한 사용자 풀에는 해킹된 자격 증명을 사용하거나 예기치 않은 위치 및 디바이스에서 로그인하는 것을 모니터링하고 방지하는 자동화된 보안 기능이 있습니다.

#### 자격 증명 풀을 사용하면 고유한 자격 증명을 생성하고 사용자에게 권한을 할당할 수 있습니다.
- 자격 증명 풀은 AWS 계정의 현재 AWS 리전에 있는 모든 자격 증명 풀 목록입니다. 자격 증명 풀 생성을 선택하여 새 자격 증명 풀 마법사를 시작합니다.
- Cognito 자격 증명 풀은 웹 및 모바일 앱에서 사용할 프로그래밍 방식의 AWS 보안 인증 정보를 생성합니다. 자격 증명 풀을 사용하면 S3, DynamoDB 및 API Gateway와 같은 AWS 서비스에서 앱 자산을 저장할 수 있습니다.
- 인증되지 않은 액세스 : 사용자가 세션을 시작하면 인증되지 않은 액세스 기능이 초기 게스트 리소스에 대한 액세스 권한을 생성합니다.
- 인증된 액세스 : 사용자에게 앱에 로그인하고 Cognito 사용자 풀을 비롯한 소셜 또는 엔터프라이즈 ID 제공업체(IdP) 중 하나를 통해 인증하도록 요청할 수 있습니다.
- 앱 권한 : 인증된 사용자의 속성에 따라 웹 보안 인증 정보로 임시 세션을 생성하고 고유한 속성을 기반으로 권한을 할당할 수 있습니다. 사용자의 자격 증명 세부 정보가 변경되는 경우 사용자 권한도 함께 변경할 수 있습니다.
